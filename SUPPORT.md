# Getting Support

Thanks for using Ratchet! Here's how to get help.

## Where to go

| I want to… | Go here |
|---|---|
| Ask a question / share what your agent learned | **GitHub Discussions** |
| Report a bug | **GitHub Issues** → Bug report |
| Report my agent learned something wrong | **GitHub Issues** → Skill / learning issue |
| Request a feature | **GitHub Issues** → Feature request |
| Report a security vulnerability | **Do not open a public issue** — see [SECURITY.md](SECURITY.md) |
| Chat with the community | Discord *(coming soon)* |

## Before you open an issue

1. Run `ratchet doctor` and include the output — it captures your version,
   config (with secrets redacted), and environment.
2. Search existing issues and discussions; someone may have hit it already.
3. For a learning/skill problem, include the relevant **ProofRun id** and the
   vault note region if you can (redact anything private — Ratchet redacts by
   default, but double-check).

## Scope of support

- **In scope:** installing and running Ratchet, configuring providers/vault,
  understanding why a skill was promoted or rejected, bug reports.
- **Out of scope:** debugging your host agent or model provider itself, and
  general "write my app" requests. We're happy to point you in the right
  direction, though.

## Response expectations

This is an early, community-driven project maintained in the open. We aim to
acknowledge issues within a few days, but please be patient — and the best way
to get something fixed fast is a clear repro or a PR. Early contributors shape
the whole thing.
